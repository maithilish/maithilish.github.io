Type Parameter Bound

The List<E> can take any type as type argument. With Type Parameter Bound we can restrict accepted types.

    public class FoodList<T extends Food> {

        public void add(T element) {
        }

        public T get(int index) {
            // return element
        }
    }

    FoodList<Food> foods = new FoodList<>();
    FoodList<Cookie> cookies = new FoodList<>();
    FoodList<Apple> apples = new FoodList<>();

    FoodList<String> apples = new FoodList<>();     // error

The FoodList accepts Food or any of its subtype as T extends Food, but any other type is not accepted.

Member Visibility

First let’s consider type which is not bounded.

    public class List<T> {
        public void someMethod(T element){
            // on element, we can access only the members (fields and methods)
            // of Object class such as getClass() etc.,
            element.getClass();
        }
    }

The actual type that replaces T can be anything and only thing that is known is that it is subtype of Java Object class. Because of this, we can only access methods such as getClass(), hashCode() and equals() etc., of Object class on element variable.

Now consider,

    public class SnackList<T extends Snack> {
        public void someMethod(T element){            
             // we can access members of Snack and its supertypes - Food and Object.             
             element.snackMethod();
             element.foodMethod();
             element.getClass();

             element.cookieMethod(); // error
        }        
    }

By extending T with Snack, all methods of Snack and its supertypes - Food and Object are visible.

There is a small twist here.

    SnackList<Snack> snacks = new SnackList<>();
    SnackList<Cookie> cookies = new SnackList<>();

    SnackList<Food> foods = new SnackList<>();  // error

We can create SnackList of Snack or Cookie but not Food as <T extends Snack> allows only Snack and its subtypes. But, within the SnackList we can access members of Snack, Food and Object but not Cookie.

Recursive Type Bound

A type parameter bounded by an expression may contain type parameter itself. This is known as recursive type bound. The max() method from javax.util.Collection which used recursive type bound is shown below:

   public static <T extends Comparable<T>> T max(List<T> list) { ... }

The type parameter of max() is <T extends Comparable<T>> which is read as “for every type T that can be compared to itself”. In other words, every element of list should be comparable with each other.

Summary

  • the <T extends type> restricts the generic type to type and its subtype.
  • type parameter bound also make members of type and all its supertypes visible within the generic type.
  • there is no type parameter bound <T super type>.
  • recursive type bound <T extends Comparable<T>> is read as “for every type T that can be compared to itself”.

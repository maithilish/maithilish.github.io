Facebook App ID for Share on Social

First we need to register as developer with Facebook. Open Facebook Developers and sign in with your Facebook account. Clicks Apps in top menu bar and select Register as Developer from the drop down menu. Complete the developer registration process by PIN verification via SMS or Phone call. After registering as developer go back to Facebook Developers and select Add a New App from Apps drop down menu.

Click Add New App from Apps drop down menu.[/caption]

In New App popup, select Website. In the next popup, Quick start for Website, enter a unique name for your App and click Create New Facebook App ID button.

Enter unique ID for your App.

In Create a New App ID popup, choose a category for your app and click Create App ID button.

Choose a category for your app.

Facebook will create the App ID and show App Configuration setup. Enter URL of your WordPress website in both Site URL and Mobile Site URL fields and click next.

Enter URL of your Wordpress site in both fields.

Now App ID is created. Next we need to configure the app. Click Apps in top bar and drop down menu now shows the newly created app id. Select the new App ID which displays the Dashboard of the Facebook App. Select Settings from left menu bar to open the Settings page.

In Settings Page, enter your site URL in App Domain and also enter Email ID.

In Settings Page, enter URL of your Wordpress site in App Domains fields and also enter your email in Contact Email field. Email is mandatory to change the app status to Public in the next step.

In Status and Review Page, toggle button to Yes to make it available to Public.

Select Status and Review from menu to open Status page. Toggle Do you want to make this app available to general public to Yes.

In App Details Page, enter tag line and description of your app.

Finally select App Details from menu to open App Info page where you need to enter tag line and description for the app.

We are done, now go to Dashboard of the App and note down App ID which you can use with Share on Social Wordpress Plugin.

Google Plus Client ID for Share on Social

Sign in to Google Developer Console with your Google account. Next, we need to create a project to hold our client app. Click Create Project

Create a new Project to hold the app.

In New Project popup, give a name to your project.

Give a name to the Project.

Console will show the Project dashboard with menu bar on left.

Google Developers Console Project Dashboard.

Next we need to enable Google+ API for the project. To do that, expand APIs and Auths menu and select APIs. By default, some Cloud and Big Query API are enabled. We need to scroll down and go to line item Google+ API and toggle Status from Off to On. Now Google+ API get listed in Enabled APIs section.

Enable Google+ API.

With that we are ready to create the Client ID. Select Credentials from the menu bar. Click Create new Client ID button displayed under OAuth.

Create new Client ID.

In Create Client ID popup, choose the Application type as Web application and click Configure Consent Screen button

Select Application Type.

In Consent Screen, enter your Email id and the product name of your site.

In consent screen, enter the Email ID and some product name.

On Accept, you will be back in Create Client ID popup where you need to enter your Wordpress site URL in Authorized Javascript Origins field. Remove any URL listed Authorized Redirect URIs field. Click Create Client ID to create the client id.

In Javascript origins enter your Wordpress site URL.

Now Credentials Page displays the details of the new created Client Id.

Copy the Client ID to use with Share on Social.

Copy the Client ID and use it in Share on Social Settings.
